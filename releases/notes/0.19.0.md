## Highlights
- The resident idle Island is lighter on energy: low-value global mouse movement tracking is reduced, secondary mascot previews stay static, and quiet-period watcher discovery and sync work back off more aggressively.
- Codex usage reads are lighter: rollout logs are no longer repeatedly reparsed when app-server snapshots already provide the needed data, reducing long-running disk I/O and CPU hotspots.
- Settings analytics now show estimated spend and a 30-day token trend alongside local Agent activity.
- Qwen Code shell approvals can now persist for the session, so repeated shell permission prompts can reuse the existing choice.

## Fixes
- Fixed quiet visible-state paths that still behaved like active UI, reducing the energy and I/O pressure reported in issue `#210`.
- Fixed Codex fallback syncs that could rescan logs frequently even when snapshots were available.
- Added regression coverage for low-power policy, Codex usage loading, watcher backoff, Qwen hook payloads, and analytics cost estimation.

## Notes
- The Sparkle build number is now `52`, and the app version is now `0.19.0`.
- Cost display is an estimate based on local token deltas, not a provider invoice.
- This is a latest-`main` performance and observability release focused on CPU, disk I/O, and energy behavior during long-running resident use.
